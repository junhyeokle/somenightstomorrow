Phase 0 — Foundation

Know why you build.

Goal

프로젝트를 시작하기 전에 방향과 철학을 먼저 세운다.

Tasks

* GitHub Repository 생성
* 프로젝트 기본 문서 구조 설계
* FOUNDING.md 작성
* NON_NEGOTIABLES.md 작성
* README.md 작성
* IDEAS.md 작성
* days/ 기록 시스템 구축
* build_logs/ 기록 시스템 구축
* Git 운영 방식(Commit / Push 전략) 정리

Success Criteria

프로젝트의 목적과 철학을 명확하게 설명할 수 있으며, 앞으로의 개발과 기록을 위한 기반이 모두 준비된 상태가 된다.

⸻

Phase 1 — Planning & Design

Draw before you build.

Goal

코드를 작성하기 전에 사이트의 구조와 방향을 명확하게 설계한다.

Tasks

* Site Map 설계
* 각 페이지의 역할 정의
* Home 구조 설계
* 콘텐츠 구조 설계
* 디자인 철학 정리
* 색감 및 분위기 결정
* 애니메이션 원칙 정의
* 공통 컴포넌트 설계

Success Criteria

사이트를 만들기 전에 어떤 페이지가 존재하는지, 각 페이지가 어떤 역할을 하는지, 어떤 경험을 제공하는지 설명할 수 있다.

디자인을 구현하지 않아도 사이트의 전체 구조를 다른 사람에게 설명할 수 있는 상태가 된다.



Phase 2 — Build

Turn ideas into reality.

Goal

설계한 구조를 실제 웹사이트로 구현한다.

Tasks

* Next.js 프로젝트 생성
* Tailwind CSS 설정
* 공통 Layout 구성
* 공통 컴포넌트 개발
* Home 페이지 구현
* About 페이지 구현
* Projects 페이지 구현
* Archive 페이지 구현
* Journal (days) 페이지 구현

Success Criteria

사이트의 기본 구조가 완성되고 모든 페이지를 이동하며 사용할 수 있다.

⸻

Phase 3 — Content

Give the project a voice.

Goal

빈 웹사이트를 나만의 이야기와 프로젝트로 채운다.

Tasks

* About 내용 작성
* Timeline 작성
* Projects 내용 작성
* Archive 작성
* Journal 이전 기록 정리
* 대표 이미지 및 콘텐츠 추가

Success Criteria

웹사이트를 방문한 사람이 나와 프로젝트를 이해할 수 있을 정도의 콘텐츠가 준비된다.

⸻

Phase 4 — Connect

Make everything work together.

Goal

정적인 웹사이트를 실제 서비스처럼 연결한다.

Tasks

* 데이터 구조 설계
* 백엔드 연결
* Markdown 연동
* GitHub API 연동
* 자동화 기능 구현
* 검색 및 필터 기능 구현

Success Criteria

콘텐츠를 쉽게 추가하고 관리할 수 있으며, 사이트가 동적인 데이터와 연동된다.

⸻

Phase 5 — Beta

Let people experience it.

Goal

실제 사용자에게 공개하고 피드백을 받는다.

Tasks

* 첫 배포
* 친구 및 주변 사람들에게 공유
* 피드백 수집
* 버그 수정
* UX 개선

Success Criteria

실제 사용자 피드백을 받고 이를 반영하여 사이트를 개선한다.

⸻

Phase 6 — Release

Keep building.

Goal

첫 번째 정식 버전을 공개하고 지속적으로 발전시킨다.

Tasks

* 도메인 연결
* SEO 및 성능 개선
* 문서 정리
* 정식 공개
* 지속적인 업데이트

Success Criteria

somenightstomorrow.dev(또는 .com)에서 안정적으로 운영되며, 앞으로도 지속적으로 발전할 수 있는 기반을 갖춘다.
